Archicad Building and MEP Automation Workbench

MEPbridge ACAIstr (MACAI) is an Archicad-native Add-On and local MCP / Workbench collaboration system. Its initial focus is to establish safe, controlled, bidirectional interaction between large language models and professional 3D design software, providing natural-language invocation, structured command execution, and autonomous AI Loop capabilities for design workflows. Through Archicad's official JSON API, the MEPBridge.* structured command interfaces, and its safety-controlled execution mechanism, AI can read, create, modify, and validate real 3D models under user confirmation and risk controls, progressively exploring practical paths toward intelligent, multidisciplinary, end-to-end building design workflows.

1Install and Start

  1. Download the full package for your language, then extract MEPbridge-ACAIstr-v0.1.0-win64-<locale>.zip to a normal writable directory. Use en-US for English or zh-CN for Chinese.
  2. Close Archicad, double-click Install-MEPBridge.cmd in the extracted package root, and allow Windows administrator permission. The installer copies the APX that matches the detected Archicad version and writes the local configuration required for one-click menu startup.
  3. Restart Archicad.
  4. In Archicad, choose MEPbridge ACAIstr -> Open MEPbridge ACAIstr. The Add-On checks and starts the local Workbench Server, then opens the Workbench page. No separate command window is required.

Manual Server fallback

Use this only when menu startup fails or when Server logs are needed:

  1. Open the release package root in File Explorer. This is the directory containing server, tools, and the README files.
  2. Click the address bar, type powershell, and press Enter. You can also right-click an empty area and choose "Open in Terminal."
  3. Run:
    node server\server.js
  4. Keep the terminal open and browse to http://127.0.0.1:19780/. Press Ctrl+C to stop the Server.

If node is not recognized, install Node.js first. The Server binds to 127.0.0.1 by default. The Archicad "User Guide" menu command also starts the Server before opening this page.

For the first validation, use a test project. Begin with Ping, selection, or project-information reads before creating, moving, editing, or deleting model elements.

2Workbench Modes and Workflow

Manual BASE

  1. Use explicit commands, parameter entry, selection operations, and debugging.
  2. The middle workspace includes QUERY, MODIFY, BUILDING, Water, Electrical, and Ventilation modules.
  3. Basic flow: choose a module, select a command, enter parameters, confirm, execute, and inspect readback.

AI Copilot

  1. Use natural-language requests, template tasks, and multi-step execution plans.
  2. Basic flow: enter a goal, generate a plan, confirm according to risk, execute steps, and inspect readback and audit results.
  3. For long plans, the middle execution panel keeps a fixed height and supports mouse-wheel or scrollbar navigation.

Without an LLM

  1. Manual BASE, fixed templates, and configured custom natural-language commands remain available.
  2. Free-text requests that require model interpretation are unavailable.

3Viewport and Execution Results

Real AABB 2D representation

  1. After Archicad returns selection or command readback, the viewport uses AABB coordinates and dimensions to show position, extent, and relative scale.
  2. When the same GUID is moved or resized, the viewport recalculates the scale from the new coordinates.
  3. An AABB is a bounding-box representation; it does not show rotated surfaces, openings, stair treads, or complex polygon detail.

Plan preview and native screenshot

  1. When no Archicad geometry readback is available, Copilot shows only a planned workflow preview, not real model geometry.
  2. Use the screenshot view to inspect element appearance. A native screenshot is closer to the actual visual result than the AABB view.

To determine whether a command actually succeeded, prioritize the execution response, GUIDs, real readback, and the Archicad model. Do not rely on the plan preview alone.

5MCP and API Integration

MCP clients

  1. The MCP Server dynamically generates 59 tools from ai-adapter\tool-descriptors.json.
  2. Start the Workbench Server before using MCP.
{ "mcpServers": { "mepbridge": { "command": "node", "args": ["D:\\MEPbridge-ACAIstr\\tools\\mepbridge-mcp-server.js"], "env": { "MEPBRIDGE_ENDPOINT": "http://127.0.0.1:19780" } } } }

Configuration formats vary among Claude Desktop, Cursor, Codex Desktop, and CodeBuddy. See INSTALL.md in the release package for complete examples.

Server API

  1. The UI and MCP enter through the Server's /api/execute endpoint, which then calls Archicad's official JSON API.
  2. Common endpoints include /health, /api/ping, /api/execute, /api/copilot/message, and the user-asset endpoints.
  3. The Archicad JSON API is detected dynamically on local ports 19723-19743; do not assume a single fixed port.

6Safety, Data, and Privacy

  1. Before write, move, edit, batch, or delete operations, verify target GUIDs, the current selection, units, stories, and coordinates.
  2. When a command interface supports dryRun, use it first to preflight structure and parameters. A dry-run does not mean that Archicad created or changed an element.
  3. When the UI requests confirmation, review the actual impact scope. After execution, verify the readback and the Archicad model.
  4. User templates, custom commands, LLM configuration, learning memory, and audit data are stored under %APPDATA%\MEPBridge by default and can be redirected with MEPBRIDGE_DATA_DIR.
  5. Do not submit API keys, user models, runtime logs, or user data to public repositories or feedback reports.

7Version Support and Known Boundaries

  1. Archicad 28 / Archicad 29: Both are released. Functional differences will be updated progressively in later versions.
  2. Complex geometry: The AABB viewport is for extent and relative-scale checks only. Use the Archicad model and native screenshots for precise geometry.
  3. Mutation operations: Validate the test release in a copy or test project and back up important models first.
