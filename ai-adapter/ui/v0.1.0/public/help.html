Archicad 建筑与 MEP 自动化工作台

MEPbridge ACAIstr（MACAI）是一套面向 Archicad 平台的原生 Add-On 与本地 MCP / Workbench 协同系统。基础阶段致力于打通大语言模型与专业三维设计软件之间安全、可控的双向交互，为设计工作提供自然语言对话调用、结构化命令执行与自主循环（AI Loop）能力。通过 Archicad 官方 JSON API、MEPBridge.* 结构化命令接口与安全执行机制，AI 可在用户确认和风险控制下读取、创建、修改并校验真实三维模型，逐步探索建筑设计全流程、全专业智能化的可落地路径。

1安装与启动

  1. 按界面语言下载完整安装包，并将 MEPbridge-ACAIstr-v0.1.0-win64-<locale>.zip 解压到普通可写目录；中文选择 zh-CN，英文选择 en-US。
  2. 关闭 Archicad，在解压后的发布包根目录双击 Install-MEPBridge.cmd，并允许 Windows 管理员权限。安装程序会复制与 Archicad 版本匹配的 APX，并写入菜单一键启动所需的本机配置。
  3. 重新启动 Archicad。
  4. 在 Archicad 菜单中点击 MEPbridge ACAIstr → 打开 MEPbridge ACAIstr。插件会自动检查并启动本机 Workbench Server，然后打开工作台页面；无需另开命令窗口。

手动启动备用方式

仅在菜单启动失败或需要查看 Server 日志时使用：

  1. 在文件资源管理器中进入发布包根目录，即能看到 server、tools 和 README 文件的目录。
  2. 点击地址栏，输入 powershell 后按 Enter；也可以右键目录空白处，选择“在终端中打开”。
  3. 在打开的 PowerShell 或终端中输入：
    node server\server.js
  4. 保持窗口开启，在浏览器打开 http://127.0.0.1:19780/；结束时按 Ctrl+C 停止 Server。

若提示无法识别 node，请先安装 Node.js。Server 默认只绑定 127.0.0.1；Archicad 菜单中的“使用指南”也会自动启动 Server 后打开本页面。

首次验证建议使用测试项目，先执行 Ping、读取选择集或项目信息等只读操作，再尝试创建、移动、编辑或删除。

2界面模式与操作流程

Manual BASE 手动模式

  1. 适合明确命令、参数录入、选择集操作和调试验证。
  2. 中区模块包括 QUERY、MODIFY、BUILDING、Water、Electrical 和 Ventilation。
  3. 基本流程：选择模块 → 选择命令 → 填写参数 → 确认 → 执行 → 查看读回。

AI Copilot 智能模式

  1. 适合自然语言请求、模板任务和多步计划执行。
  2. 基本流程：输入目标 → 生成计划 → 按风险策略确认 → 执行步骤 → 查看读回与审计结果。
  3. 步骤较多时，中区窗口保持固定高度，可用鼠标滚轮或滚动条上下查看。

无 LLM 时

  1. Manual BASE、固定模板和已配置的自定义自然语言命令仍可使用。
  2. 需要模型理解的自由文本自动解析不可用。

3视口与执行结果

实际 AABB 二维示意

  1. Archicad 返回选择集或命令读回数据后，视口按 AABB 坐标和尺寸显示位置、范围与相对比例。
  2. 同一 GUID 移动或缩放后，视口会按新坐标重新计算比例。
  3. AABB 是包围盒示意，不表达旋转面、孔洞、楼梯踏步或复杂多边形细节。

计划示意与原生截图

  1. 没有 Archicad 几何读回时，Copilot 只显示计划路径示意，不代表真实模型几何。
  2. 核对构件外观时，请切换到截图视图；截图比 AABB 更接近实际视觉结果。

判断命令是否真正生效，应优先查看执行返回、GUID、实际读回和 Archicad 模型；不能只根据计划示意判断。

5MCP 与 API 接入

MCP 客户端

  1. MCP Server 从 ai-adapter\tool-descriptors.json 动态生成 59 个工具。
  2. 使用 MCP 前，先启动 Workbench Server。
{ "mcpServers": { "mepbridge": { "command": "node", "args": ["D:\\MEPbridge-ACAIstr\\tools\\mepbridge-mcp-server.js"], "env": { "MEPBRIDGE_ENDPOINT": "http://127.0.0.1:19780" } } } }

Claude Desktop、Cursor、Codex Desktop 和 CodeBuddy 的配置格式可能不同，完整示例请查看发布包中的 INSTALL.zh-CN.md。

Server API

  1. UI 和 MCP 统一通过 /api/execute 进入 Server，再由 Server 调用 Archicad 官方 JSON API。
  2. 常用端点：/health、/api/ping、/api/execute、/api/copilot/message 和用户资产端点。
  3. Archicad JSON API 在本机 19723-19743 范围动态探测，不要固定写死单一端口。
POST http://127.0.0.1:19780/api/execute Content-Type: application/json { "commandName": "MEPBridge.Ping", "parameters": {} }

6安全、数据与隐私

  1. 写入、移动、编辑、批量和删除操作应先核对目标 GUID、当前选择集、单位、楼层和坐标。
  2. 命令接口支持 dryRun 时，优先先做结构和参数预检；dry-run 不代表已经在 Archicad 中创建或修改构件。
  3. 界面提示确认时，应确认实际影响范围；执行后使用读回结果和 Archicad 模型复核。
  4. 用户模板、自定义命令、LLM 配置、学习记忆和审计数据默认写入 %APPDATA%\MEPBridge，可通过 MEPBRIDGE_DATA_DIR 覆盖。
  5. 不要把 API Key、用户模型、运行日志或用户数据提交到公开仓库或反馈内容中。

7版本支持与已知边界

  1. Archicad 28 / Archicad 29：均已正常发布，功能区别将在后续版本中逐步更新。
  2. 复杂几何：AABB 视口仅用于范围和比例核对，精确几何仍以 Archicad 模型与原生截图为准。
  3. 写入操作：测试版建议在副本或测试项目中验证，重要模型应先备份。
